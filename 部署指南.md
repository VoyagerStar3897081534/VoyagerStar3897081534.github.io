## 🎯 方案 1：Railway 部署（推荐⭐⭐⭐⭐⭐）

**优势**：不休眠、每月 $5 免费额度、自动 HTTPS、配置简单

### 部署步骤

#### 1️⃣ 注册 Railway 账号
访问：https://railway.app
- 点击 "Login" → 选择 "Sign in with GitHub"
- 授权访问你的 GitHub 账号

#### 2️⃣ 创建新项目
1. 登录后点击 **"New Project"**
2. 选择 **"Deploy from GitHub repo"**
3. 选择你的仓库：`VoyagerStar3897081534.github.io`
4. 点击 **"Deploy"**

#### 3️⃣ 等待自动部署
Railway 会自动：
- 检测 Node.js 项目
- 安装依赖（`npm install`）
- 启动服务器（`npm start`）
- 分配域名

#### 4️⃣ 获取你的 API 域名
1. 部署完成后，点击项目卡片
2. 点击顶部的 **"Settings"** 标签
3. 找到 **"Domains"** 部分
4. 点击 **"Generate Domain"**
5. 复制生成的域名，例如：
   ```
   voyagerstar-blog-production.up.railway.app
   ```

#### 5️⃣ 更新前端配置
修改 `auth.js` 文件第 3 行：

```javascript
// 将这一行：
const API_BASE = 'http://localhost:3000/api';

// 改为你的 Railway 域名（注意添加 https://）：
const API_BASE = 'https://voyagerstar-blog-production.up.railway.app/api';
```

#### 6️⃣ 提交并推送
```bash
git add .
git commit -m "配置 Railway 部署"
git push origin main
```

Railway 会自动检测新代码并重新部署！

**优势**：完全免费、配置简单

- 点击 "Get Started for Free"

2. 选择 **"Web Service"**
3. 连接你的 GitHub 仓库
4. 配置如下：

| 配置项 | 值 |
|--------|-----|
| **Name** | `voyagerstar-blog` |
| **Environment** | `Node` |
| **Build Command** | `npm install` |
| **Start Command** | `npm start` |
| **Plan** | `Free` |

5. 点击 **"Create Web Service"**

#### 3️⃣ 获取域名
部署完成后，你会看到类似地址：
```
https://voyagerstar-blog.onrender.com
```

#### 4️⃣ 更新前端配置
修改 `auth.js`：
```javascript
const API_BASE = 'https://voyagerstar-blog.onrender.com/api';
```

## 🆘 常见问题

### Q1: Railway 的 $5 额度够用吗？
**A**: 完全够用！对于小型博客：
- 内存：512MB 足够
- 每月费用：约 $1-2
- 剩余额度可用于其他项目

### Q2: Render 休眠后影响大吗？
**A**: 
- 首次访问需要等待 30-50 秒唤醒
- 后续访问正常速度
- 适合个人博客（访问量不大）

### Q3: 数据会丢失吗？
**A**: 
- 当前使用文件系统存储（`data/` 目录）
- 数据保存在 Git 仓库中，不会丢失
- 每次部署都会保留博客文章

### Q4: 如何查看部署日志？
**Railway**: 点击项目 → "Deployments" → 选择最新部署 → "View Logs"  
**Render**: 点击项目 → "Logs" 标签

### Q5: 部署失败怎么办？
检查：
1. `package.json` 中的 `start` 脚本是否正确
2. `server.js` 是否有语法错误
3. 端口配置是否正确（使用 `process.env.PORT`）
